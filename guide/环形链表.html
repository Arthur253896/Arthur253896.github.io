# 环形链表 (opens new window)

# 1、快慢指针法：

如果用两个循环一个先走然后一个记录当前位置，那么时间复杂度为n*n，超出时间限制

# 2、两阶段

依旧是快慢指针，但改变一下策略：

1、定义两个指针slow和fast，slow每次走一格，fast每次走两格，如果两个相遇了则说明一定有环；

2、找到这个环的入口

再定义两个指针一个指向头节点index1，一个指向相遇的节点index2，入口节点就是求x；

slow走过一共x+y路程，fast走过x+n（y+z）+y，fast一次走两个，则相遇时有：2*(x+y)=x+y+n(y+z)；两边简化：

x=（n-1）*（y+z）+z;

如果n=1，则x=z说明，慢指针的路程恰好等于从相遇点到入口的距离；

如果n>1，则说明，慢指针的路程等于从相遇点到入口的距离再加n圈，

无论如何最后index1和index2相遇的位置就一定是环的入口；

当然如果fast slow没相遇则不存在环；代码如下：

    ListNode *detectCycle(ListNode *head) {
        ListNode *pre=head;
        ListNode *post=head;
        while(post!=NULL && post->next!=NULL){
            //要注意边界问题，因为post一次移动两个节点，所以要要求next不为空
            pre=pre->next;
            post=post->next->next;
            if(pre==post){
                ListNode *index1=head;
                ListNode *index2=post;//指向相遇点
                while(index1!=index2){
                    index1=index1->next;
                    index2=index2->next;
                }
                return index1;
            }
        }
        return NULL;
    }
